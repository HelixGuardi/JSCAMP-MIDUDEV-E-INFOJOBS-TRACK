## 📌 ¿Qué encontrarás aquí?

A diferencia del [repositorio oficial del bootcamp](https://github.com/midudev/jscamp), el mío está organizado **por módulos aislados**. Cada módulo contiene su propia versión del proyecto `DevJobs` en el estado exacto en que se trabajó en esa etapa. Esto permite ver la progresión lineal del código sin mezclar cambios posteriores.

## 📅 Cronología de commits

Mi regla personal: **un commit por cada clase o avance significativo**, con el siguiente formato:

`git commit -m "Módulo XX - Clase Y: descripción breve del cambio`

Así cualquiera puede navegar por el historial de GitHub y ver **exactamente qué aprendí en cada sesión**, en orden cronológico real.

## 🚀 Proyecto estrella: DevJobs

El proyecto práctico del bootcamp es **DevJobs**, una aplicación completa de búsqueda y publicación de empleos para desarrolladores.

Diseño original en [Stitch](https://stitch.withgoogle.com/projects/7508115667617706440).

En este repositorio, cada módulo contiene la versión de `DevJobs` que se desarrolló en esos momentos:

- **Módulo 00-01**: HTML/CSS y JS vainilla → maqueta interactiva.

- **Módulo 02-03**: React + estado global → SPA completa.

- **Módulo 04-05**: Node + TypeScript → API y tipado.

- ... y así sucesivamente.

Si también estás haciendo el bootcamp y quieres ver una forma diferente de organizar tu progreso:
